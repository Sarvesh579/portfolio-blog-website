## Reporting a Vulnerability

If you discover a security vulnerability, **please do not create a public issue**.

Instead, report it responsibly using one of the methods below:

### Preferred Method
- Email: **sarvesh.dabholkar41@gmail.com**

Please include:
- A clear description of the vulnerability
- Steps to reproduce (proof-of-concept if possible)
- Potential impact
- Screenshots or logs (if applicable)

## Scope of Security Considerations

This project takes reasonable precautions against common vulnerabilities, including but not limited to:

- Authentication & authorization flaws
- Input validation & sanitization
- API security (rate limiting, access control)
- Environment variable protection
- Dependency vulnerabilities

However, this project is **not intended for production-critical or public use**.
